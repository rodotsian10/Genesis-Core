# 10. AI 코딩 가이드라인 (AI Coding Guidelines & Reminders)

이 문서는 AI 어시스턴트(Antigravity)가 코드를 수정하거나 리팩토링할 때 반드시 지켜야 할 **절대적인 행동 수칙**을 정의합니다. 
AI는 작업을 시작하기 전과 완료한 후에 이 문서를 상기하여 예기치 않은 버그나 기능 누락을 방지해야 합니다.

## 🚨 절대 수칙 (CRITICAL RULES)

### 1. 무관한 코드 임의 삭제 및 축약 금지 (No Silent Deletions)
새로운 기능을 추가하거나 기존 로직을 리팩토링할 때, **현재 작업과 직접적인 관련이 없는 코드를 실수로 누락하거나 삭제해서는 안 됩니다.**
* **예시 1:** UI에 로그 기능을 추가한다고 해서 기존에 잘 렌더링되던 체력(HP) 게이지바 렌더링 코드를 누락시키면 안 됩니다.
* **예시 2:** 함수 파라미터나 시그니처를 유지해야 합니다. (`camera.set_target(entity, self.world)`에서 `self.world`를 빼먹는 실수 등)
* **예시 3:** 파일 상단의 `import math`, `import random` 등 기존 모듈 임포트를 실수로 덮어씌워 날려버리면 안 됩니다.

### 2. 전체 파일 보존 원칙 (Preserve Unrelated Sections)
파일을 덮어쓰거나(`replace_file_content` 또는 `write_to_file`) 교체할 때, 수정 지점을 제외한 모든 코드가 **토씨 하나 틀리지 않고 그대로 유지되는지 확인**해야 합니다. 
코드를 길게 쓰기 귀찮다는 이유로 `... (기존 코드 유지) ...` 형태로 임의 축약하여 실제 파일 내용이 날아가는 대참사를 만들지 마십시오.

### 3. 교차 검증 (Cross-Validation Before Completion)
코드 수정을 마친 후에는 반드시 다음 질문을 스스로 던지며 검토합니다.
* "내가 수정한 범위 밖의 코드가 온전히 남아있는가?"
* "이 수정으로 인해 호출부나 다른 모듈(import)에서 에러가 발생하지는 않는가?"
* "내가 추가한 기능 때문에 기존의 핵심 UI나 로직이 가려지거나 사라지지 않았는가?"

### 4. 새로운 클래스/컴포넌트 추가 시 Import 필수 확인 (No NameError)
새로운 기능이나 컴포넌트(예: `BrainComponent`)를 정의하고 다른 파일(`systems.py`, `main.py` 등)에서 사용할 때, **해당 파일 상단에 반드시 `import` 구문을 추가했는지 이중으로 확인해야 합니다.**
* 새로 만든 클래스를 불러다 쓰면서 import를 빼먹어 `NameError: name '...' is not defined`를 발생시키는 초보적인 실수를 절대 하지 마십시오.

## 📝 징계 사항
위 수칙을 어기고 멀쩡히 작동하던 기존 기능을 망가뜨리는 실수를 반복할 경우, 유저의 신뢰를 잃고 개발 흐름을 크게 방해하게 됨을 명심하십시오. **'한 번에 하나의 기능만 확실하게, 주변 코드는 절대 훼손하지 않는다'**는 원칙을 가슴 깊이 새기십시오.
